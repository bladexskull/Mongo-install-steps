Prerequisites

Before starting, ensure you have three Ubuntu servers with network connectivity between them.

Server 1

IP: 172.31.30.124

Hostname: mongo1

Role: Primary

Server 2

IP: 172.31.25.72

Hostname: mongo2

Role: Secondary

Server 3

IP: 172.31.19.138

Hostname: mongo3

Role: Secondary

System Requirements

  • Ubuntu 20.04/22.04/24.04 on all servers
  • Minimum 2 CPU cores per server
  • Minimum 4GB RAM per server
  • Minimum 20GB storage per server
  • SSH access to all servers
  • Sudo privileges for installation

Network Configuration

Ensure these ports are open between all servers:

  • 27017 - MongoDB default port
  • 22 - SSH (for administration)
Note: All commands in this guide should be run as root or with sudo privileges.

Update System Packages

On all servers, update the package index:

Execute on ALL servers
sudo apt update && sudo apt upgrade -y

Host Configuration

Set hostnames and configure network settings on all servers.

Set Hostnames

On each server, set the appropriate hostname:

On mongo1
sudo hostnamectl set-hostname mongo1
On mongo2
sudo hostnamectl set-hostname mongo2
On mongo3
sudo hostnamectl set-hostname mongo3

Update Hosts File

Execute on ALL servers
# Update /etc/hosts echo "172.31.30.124 mongo1" | sudo tee -a /etc/hosts echo "172.31.25.72 mongo2" | sudo tee -a /etc/hosts echo "172.31.19.138 mongo3" | sudo tee -a /etc/hosts # Verify changes cat /etc/hosts
Tip: Replace IP addresses with your actual server IPs if different.

Generate and Distribute Keyfile

The keyfile enables authentication between replica set members.

Security Note: The keyfile must be identical on all servers and have 400 permissions. This file acts as a shared password for intra-cluster authentication.

Generate Keyfile on Primary Server

mongo1 (172.31.30.124)
Generate keyfile and set permissions
# Generate keyfile sudo openssl rand -base64 756 | sudo tee /etc/mongodb-keyfile >/dev/null # Set permissions sudo chmod 400 /etc/mongodb-keyfile sudo chown mongodb:mongodb /etc/mongodb-keyfile # Verify permissions ls -l /etc/mongodb-keyfile

Copy Keyfile to Secondary Servers

Distribute keyfile to all nodes
# Copy to mongo2 sudo scp /etc/mongodb-keyfile ubuntu@mongo2:/tmp/ ssh ubuntu@mongo2 "sudo mv /tmp/mongodb-keyfile /etc/" # Copy to mongo3 sudo scp /etc/mongodb-keyfile ubuntu@mongo3:/tmp/ ssh ubuntu@mongo3 "sudo mv /tmp/mongodb-keyfile /etc/"

Set Permissions on All Servers

Execute on ALL servers
sudo chmod 400 /etc/mongodb-keyfile sudo chown mongodb:mongodb /etc/mongodb-keyfile # Verify on each server ls -l /etc/mongodb-keyfile

Install MongoDB v8.0.10

Install MongoDB 8.0 on all three servers.

Note: We're starting MongoDB temporarily without security to set up the replica set. We'll enable security after initialization.

Import MongoDB GPG Key

Execute on ALL servers
# Install prerequisites sudo apt-get install -y gnupg curl # Add MongoDB repository key curl -fsSL https://www.mongodb.org/static/pgp/server-8.0.asc | \ sudo gpg -o /usr/share/keyrings/mongodb-server-8.0.gpg --dearmor

Add MongoDB Repository

Ubuntu 22.04
Ubuntu 20.04
Ubuntu 24.04
# For Ubuntu 22.04 (Jammy) echo "deb [ arch=amd64,arm64 signed-by=/usr/share/keyrings/mongodb-server-8.0.gpg ] \ https://repo.mongodb.org/apt/ubuntu jammy/mongodb-org/8.0 multiverse" | \ sudo tee /etc/apt/sources.list.d/mongodb-org-8.0.list
# For Ubuntu 20.04 (Focal) echo "deb [ arch=amd64,arm64 signed-by=/usr/share/keyrings/mongodb-server-8.0.gpg ] \ https://repo.mongodb.org/apt/ubuntu focal/mongodb-org/8.0 multiverse" | \ sudo tee /etc/apt/sources.list.d/mongodb-org-8.0.list
# For Ubuntu 24.04 (Noble) echo "deb [ arch=amd64,arm64 signed-by=/usr/share/keyrings/mongodb-server-8.0.gpg ] \ https://repo.mongodb.org/apt/ubuntu noble/mongodb-org/8.0 multiverse" | \ sudo tee /etc/apt/sources.list.d/mongodb-org-8.0.list

Install MongoDB Packages

Execute on ALL servers

Enable and Start MongoDB

Execute on ALL servers
sudo systemctl enable mongod sudo systemctl start mongod sudo systemctl status mongod

Configure MongoDB

Configure MongoDB on all servers to enable replication and authentication.

Edit MongoDB Configuration

Configuration for ALL servers (/etc/mongod.conf)
# Network settings net: port: 27017 bindIp: 0.0.0.0 # Listen on all interfaces # Security settings security: authorization: enabled keyFile: /etc/mongodb-keyfile # Replication settings replication: replSetName: rs0 # Replica set name # Storage settings storage: dbPath: /var/lib/mongodb journal: enabled: true # System log settings systemLog: destination: file logAppend: true path: /var/log/mongodb/mongod.log

Apply Configuration Changes

Execute on ALL servers
sudo systemctl restart mongod sudo systemctl status mongod tail -f /var/log/mongodb/mongod.log

Important: After enabling authentication, you won't be able to access MongoDB without valid credentials. We'll create these in the next step.

Initialize Replica Set

Create admin user and initialize the replica set on the primary node.

Create Admin User

mongo1 (172.31.30.124)
Create admin user
mongosh --port 27017 use admin db.createUser({ user: "admin", pwd: "M0S#sT@P@w@d11", // Replace with secure password roles: [{ role: "root", db: "admin" }] })

Initialize Replica Set

Method 1: Initialize all at once (recommended)
mongosh --port 27017 -u admin -p 'M0S#sT@P@w@d11' --authenticationDatabase admin --eval 'rs.initiate({ _id: "rs0", members: [ { _id: 0, host: "mongo1:27017" }, { _id: 1, host: "mongo2:27017" }, { _id: 2, host: "mongo3:27017" } ] })'
Method 2: Initialize and add members separately
# Initialize replica set mongosh --port 27017 -u admin -p 'M0S#sT@P@w@d11' --authenticationDatabase admin --eval 'rs.initiate()' # Add members mongosh --port 27017 -u admin -p 'M0S#sT@P@w@d11' --authenticationDatabase admin --eval 'rs.add("mongo2:27017")' mongosh --port 27017 -u admin -p 'M0S#sT@P@w@d11' --authenticationDatabase admin --eval 'rs.add("mongo3:27017")'

Check Replica Set Status

Verify cluster status
mongosh --port 27017 -u admin -p 'M0S#sT@P@w@d11' --authenticationDatabase admin --eval 'rs.status()'
Expected Output: One PRIMARY and two SECONDARY members with "health": 1
Success: Your MongoDB replica set is now initialized and ready for use.

Create Application User

Create a dedicated user for application access.

mongo1 (172.31.30.124)

Create Application User

Run in MongoDB shell
mongosh --port 27017 -u admin -p 'M0S#sT@P@w@d11' --authenticationDatabase admin use myappdb db.createUser({ user: "appUser", pwd: "SecureAppPassword456", // CHANGE THIS roles: [ { role: "readWrite", db: "myappdb" } ] })
Success: Application user created with restricted privileges.

Verification

Verify that your replica set is working correctly with authentication.

Connect with Authentication

mongosh "mongodb://admin:M0S#sT@P@w@d11@mongo1:27017,mongo2:27017,mongo3:27017/?replicaSet=rs0&authSource=admin"

Test Write Operations

Run in MongoDB shell
use testdb db.testcoll.insertOne({ message: "Replica set is working!", timestamp: new Date(), server: "Primary node" }) db.testcoll.find()

Connection String for Applications

mongodb://appUser:SecureAppPassword456@mongo1:27017,mongo2:27017,mongo3:27017/myappdb?replicaSet=rs0&authSource=admin
Congratulations! You now have a fully functional MongoDB replica set with authentication.
Maintenance Tips:
  • Regularly back up your data: mongodump --uri="mongodb://admin:password@mongo1:27017/admin?replicaSet=rs0"
  • Rotate keyfile every 6 months
  • Monitor replica set health with rs.status()
  • Keep MongoDB updated: sudo apt update && sudo apt upgrade mongodb-org
  • Set up monitoring with MongoDB Atlas or Ops Manager
